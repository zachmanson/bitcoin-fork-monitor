```python
class EventBus:
    def set_loop(self, loop: asyncio.AbstractEventLoop) -> None: ...
    def subscribe(self) -> asyncio.Queue: ...          # returns a per-client queue
    def unsubscribe(self, q: asyncio.Queue) -> None: ...
    def notify(self, data: Any) -> None: ...           # thread-safe broadcast

event_bus = EventBus()  # module-level singleton
```

FastAPI 0.135 SSE API (available after Plan 01 upgrade):
```python
from fastapi.sse import EventSourceResponse, ServerSentEvent
from collections.abc import AsyncIterable

# Endpoint returns an async generator of ServerSentEvent objects.
# FastAPI wraps it in EventSourceResponse automatically.
@router.get("/api/events", response_class=EventSourceResponse)
async def sse_events(request: Request) -> AsyncIterable[ServerSentEvent]:
    q = event_bus.subscribe()
    try:
        while True:
            if await request.is_disconnected():
                break
            try:
                data = await asyncio.wait_for(q.get(), timeout=15.0)
                yield ServerSentEvent(data=data, event="update")
            except asyncio.TimeoutError:
                yield ServerSentEvent(comment="keepalive")
    finally:
        event_bus.unsubscribe(q)
```

    Endpoint contract:
    - GET /api/events returns HTTP 200 with Content-Type: text/event-stream
    - On each event received from the queue: yields ServerSentEvent(data=payload, event="update")
    - On 15s timeout (no events): yields ServerSentEvent(comment="keepalive") to keep connection alive
    - On client disconnect (request.is_disconnected()): breaks the loop
    - In finally block: always calls event_bus.unsubscribe(q)
  </behavior>

    1. Create app/routers/events.py:
       - APIRouter(prefix="/api", tags=["events"])
       - Imports: asyncio, from collections.abc import AsyncIterable, from fastapi import APIRouter Request,
         from fastapi.sse import EventSourceResponse ServerSentEvent, from app.events import event_bus
       - GET /api/events endpoint:
           @router.get("/api/events", response_class=EventSourceResponse)
           async def sse_events(request: Request) -> AsyncIterable[ServerSentEvent]:
       - Subscribe on entry: q = event_bus.subscribe()
       - Wrap everything in try/finally with event_bus.unsubscribe(q) in finally
       - Inner loop: check await request.is_disconnected() first (break if True), then
         await asyncio.wait_for(q.get(), timeout=15.0), yield the event. Catch TimeoutError
         and yield keepalive comment.
       - Add a docstring explaining the key concepts:
         - Why SSE instead of WebSocket: SSE is unidirectional (server → browser), simpler
           for broadcast-only use cases, no client library needed (built into browsers as EventSource).
         - Why per-client queue: each browser tab has its own queue so events are not stolen
           by other consumers.
         - Why asyncio.wait_for with timeout: SSE connections are long-lived; without a timeout,
           an idle connection would block indefinitely. The keepalive comment prevents
           proxies/browsers from timing out the connection.
         - Why request.is_disconnected(): Without this check, closed browser tabs leave their
           queues consuming memory until the process restarts.
