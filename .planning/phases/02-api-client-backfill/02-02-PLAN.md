Pagination pattern (from RESEARCH.md):
The API returns blocks DESCENDING from startHeight. To walk ascending from resume_height to tip:
- Each page: call fetch_blocks_page(current_height + 14) to get blocks [current_height .. current_height+14]
- Advance: current_height += 15
- Stop when current_height > tip_height
The tip_height is the height of the first block on the first page fetched (before the loop, call
fetch_blocks_page with a very large number or current highest known height to get the tip).

Simpler tip-detection: fetch one page with a large sentinel height to find the current tip.
The API returns the most recent blocks when startHeight exceeds the current chain tip.
fetch_blocks_page(999_999_999) returns the current tip page. blocks[0]["height"] is the current tip.

    logger = logging.getLogger(__name__)

    @asynccontextmanager
    async def lifespan(app: FastAPI):
        # Startup
        create_db_and_tables()

        with Session(engine) as session:
            state = session.exec(select(SyncState)).first()
            already_done = state is not None and state.backfill_complete

        if already_done:
            logger.info("Backfill already complete — skipping thread launch")
            backfill_thread = None
        else:
            logger.info("Starting backfill worker thread")
            backfill_thread = threading.Thread(
                target=run_backfill,
                daemon=True,
                name="backfill"
            )
            backfill_thread.start()

        yield  # FastAPI serves requests here

        # Shutdown
        if backfill_thread is not None and backfill_thread.is_alive():
            logger.info("Waiting for backfill thread to finish (timeout=5s)...")
            backfill_thread.join(timeout=5.0)

    app = FastAPI(title="Bitcoin Fork Monitor", lifespan=lifespan)

    # Minimal health endpoint to confirm the server is up
    @app.get("/health")
    def health() -> dict:
        """Return server status. Used to confirm startup succeeded."""
        return {"status": "ok"}
